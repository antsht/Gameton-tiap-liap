1. Основной цикл управления (

loop
 и

processTurn
)
func (b *Bot) loop()
Главный бесконечный цикл бота. Он работает в отдельной горутине.

lastProcessedTurn: Переменная, хранящая номер последнего обработанного хода. Нужна, чтобы не посылать команды несколько раз на одном и том же ходу.
select { case <-b.stopCh: return ... }: Проверка, не пришел ли сигнал на остановку бота.
arena, err := b.client.GetArena(): Запрос текущего состояния игры (карты) у сервера.
b.state.ArenaLock.Lock(): Блокировка мьютекса для безопасного обновления общего состояния (чтобы веб-интерфейс не прочитал «битые» данные, пока мы их пишем).
waitMs: Рассчитывает время ожидания до следующего хода (на основе данных от сервера NextTurnIn) плюс небольшой запас в 50мс.
func (b *Bot) processTurn(arena *api.PlayerResponse)
Точка входа для принятия решений на каждом конкретном ходу.

cmd (api.PlayerCommand): Объект, в который мы записываем все наши решения (апгрейды, переезд ЦУ, действия плантаций).
upgrade := b.chooseBestUpgrade(arena): Решает, какой апгрейд купить прямо сейчас.
Блок «CU Evacuation & Survival»: Самая критичная часть. Если Главная Плантация (ЦУ) в опасности (землетрясение или мало HP), бот ищет соседнюю клетку, куда можно экстренно «переехать» (RelocateMain).
eqImminent: Флаг приближающегося землетрясения (через 0-2 хода).
cuNeedsEqEscape: Нужно ли убегать (есть ли у нас иммунитет на этот ход).
b.computeHiveMind(arena): Вызов сердца стратегии для распределения мелких команд.
go b.dumpTurn(...): Асинхронное сохранение состояния хода в логах для последующего анализа (не тормозит основной цикл).
2. Структуры данных для экономики и планирования
type target struct (Цель)
baseVal: Базовая ценность цели (например, ремонт ЦУ важнее, чем охота на бобра).
usage / maxUsage: Сколько рабочих (плантаций) уже назначено на эту клетку и каков лимит.
flat: Если true, то эффективность не падает при добавлении новых рабочих (используется для стройки).
func (t *target) marginalGain(...)
Рассчитывает, сколько «очков пользы» принесет еще один рабочий, направленный на эту цель, с учетом расстояния (chainPenalty) и штрафов за усталость.

3. Симуляция будущего (Beam Search)
Бот не просто ходит, он «представляет» будущее на 3 хода вперед.

type simState struct
Упрощенная копия мира для симуляции.

plants: Список координат наших плантаций.
constructions: Текущий прогресс строек.
score: Накопленные очки за время симуляции.
func (s *simState) advance(buildCmds ...)
Продвигает виртуальный мир на 1 ход вперед:

Начисляет очки за терраформацию.
Применяет команды строительства.
Превращает завершенные стройки в новые плантации.
4. Распределение ресурсов (

computeHiveMind
)
Это «Генеральный штаб». Здесь формируется список всех возможных дел на карте.

Категории целей (targets):

Repair CU: Ремонт главного здания (приоритет 10,000).
Finish Construction: Завершение начатых строек (приоритет 3,000–15,000).
Hunt Beaver: Охота на бобров (приоритет 2,000).
Sabotage Enemy: Атака врагов (рассчитывается в

sabotageScore
).
BRIDGE REPAIR: Ремонт «мостиков» — плантаций, которые соединяют разные части вашей колонии. Если они умрут, сеть разорвется.
PREEMPTIVE BRIDGE: Если бот видит, что клетка-мост скоро деградирует (по таймеру клетки), он заранее начинает строить обходной путь.
Reconnect: Если часть плантаций оказалась изолирована (IsIsolated), бот бросает все силы на постройку соединяющей клетки.
Expansion: Обычное расширение колонии.
Механика выбора лучшего плана: Бот создает несколько «кандидатов» на стратегию:

baseline: Баланс всего.
defensive: Приоритет ремонту и выживанию.
aggressive_expansion: Приоритет росту.
vicious: Приоритет атаке врагов и бобров.
Затем он прогоняет симуляцию для каждого кандидата и выбирает того, кто через 3 хода даст лучший totalScore (больше плантаций + компактность сети).

5. Функции оценки (Scoring)
func (b *Bot) buildScore(...)
Рассчитывает, выгодно ли строиться в конкретной клетке:

cellValue: Бонус +50% для клеток на координатах, кратных 7 (усиленные узлы).
chainPenalty: Штраф за расстояние от ЦУ (чем дальше, тем дороже связь).
overbuiltRatio: Снижает приоритет стройки, если мы почти достигли лимита населения.
neighbors (Redundancy Bonus): Бот получает бонус +3000 за каждого соседа. Это заставляет его строить «сетки» и «кольца» вместо длинных тонких линий, которые легко разорвать.
func (b *Bot) sabotageScore(...)
Если у врага HP < 10, приоритет взлетает до 5000 (убить, чтобы не мучился и освободил место).
6. Система апгрейдов (

chooseBestUpgrade
)
Бот покупает улучшения строго по приоритету:

Экстренно: earthquake_mitigation, если на небе тучи и скоро тряханет.
settlement_limit: Самое важное — расширение лимита зданий.
signal_range: Дальность действий (чтобы бить дальше и строить дешевле).
max_hp: Чтобы здания жили дольше.
Все остальное по списку.
7. Вспомогательные функции (Helpers)

isOccupied
: Проверяет, занята ли клетка (горами, врагом, нами или стройкой). Огромный список проверок, чтобы не пытаться строиться на голове у врага.

manhattanDistance
: Дистанция «манхэттенское расстояние» (сумма разностей X и Y) — основная метрика расстояния в игре.

distSq
: Квадрат Евклидова расстояния (используется для расчета радиуса песчаных бурь).
Итог каждой строчки: Ваша программа — это цикл «Вижу -> Оцениваю выгоду каждой клетки -> Симулирую лучший вариант на 2-3 хода вперед -> Посылаю пачку команд». Концепция "Bridge Repair" и "Beam Search" делают этого бота очень устойчивым к разрывам сети.
